## 2026-05-25 - 예약 기록/메뉴 운영 UX 수정

- 사용자 요청 요약: `전체 기록`은 진행 중과 처리 기록이 한 화면 안에서 나뉘어 보이는 이전 방식이 더 좋고, 메뉴 운영의 품절/숨김 체크박스는 더 크게 보이며 저장 버튼 없이 즉시 적용되길 원함.
- 진행한 문제: 예약 목록의 전체 기록 표시 방식과 메뉴 운영 저장 흐름을 조정.
- 확인한 원인과 단서: 전체 기록 탭이 모든 상태를 하나의 테이블에 섞어 보여 상태 흐름이 덜 명확했고, 메뉴 운영은 체크 후 별도 저장 버튼을 눌러야 해서 조작감이 번거로웠다.
- 시도한 해결 과정: 예약 탭 컴포넌트에서 전체 기록일 때만 `진행 중`과 `처리 기록` 테이블을 분리하고, 메뉴 운영 체크 변경 시 바로 설정 API에 PATCH 하도록 변경했다.
- 적용한 변경 사항: 메뉴 운영의 품절/숨김 체크박스 크기와 터치 영역 확대, 메뉴 운영 페이지 저장 버튼 숨김, 체크 즉시 저장 메시지 표시.
- 검증 결과: `npx tsc --noEmit` 통과.
- 남은 위험 요소: 체크 즉시 저장은 네트워크가 느릴 때 잠깐 비활성화되므로 실제 브라우저에서 클릭 반응감을 확인할 필요가 있다.
- 다음 단계: 로컬 서버 재시작 후 `/admin/reservations`, `/admin/menu`에서 화면 배치와 즉시 저장 동작을 확인한다.

## 2026-05-25 - 예약 완료 문구/번호 표시 개선

- 사용자 요청 요약: 예약 완료 후 UUID가 그대로 보이지 않고 `001` 같은 짧은 예약번호로 보이길 원했고, 완료 문구를 더 부드럽게 변경 요청.
- 진행한 문제: 예약 API 응답에 사용자용 예약번호를 추가하고, 예약 완료 카드의 문구와 줄바꿈 표시를 수정.
- 확인한 원인과 단서: 내부 예약 ID는 UUID로 관리되고 있었고, 프론트 완료 화면에서 이를 그대로 출력하고 있었다.
- 시도한 해결 과정: 저장 구조는 유지하고, 예약 생성 후 같은 날짜의 생성 순서를 계산해 `001` 형식으로 내려주도록 했다.
- 적용한 변경 사항: `reservationNumber` API 응답 추가, 완료 화면 예약번호 표시 변경, 문구를 `예약 요청이 접수되었습니다 : )`로 변경.
- 검증 결과: `npx tsc --noEmit` 통과.
- 남은 위험 요소: 동일 시각 동시 예약이 극단적으로 겹치면 표시 번호 계산이 완전한 DB 시퀀스만큼 엄격하지는 않다.
- 다음 단계: 브라우저에서 새 예약을 넣어 완료 카드에 `001` 형식 번호와 줄바꿈 문구가 보이는지 확인한다.

## 2026-05-25 - PWA/Web Push 알림 테스트 구조 추가

- 사용자 요청 요약: 문자 API 없이 사이트/PWA만으로 휴대폰 예약 알림을 실험할 수 있는지 확인하고, 가능한 부분은 끝까지 구현 요청.
- 진행한 문제: 관리자 휴대폰에서 웹 푸시 구독을 등록하고 새 예약 생성 시 알림을 보내는 기본 구조를 추가.
- 확인한 원인과 단서: 배포 주소가 HTTPS라 PWA/Web Push의 필수 조건은 충족되어 있었고, 문자 API 없이도 VAPID 기반 웹 푸시로 실험 가능했다.
- 시도한 해결 과정: `web-push` 패키지 설치, VAPID 키 생성, 로컬 `.env`에 키 저장, manifest/service worker/API/provider/관리자 버튼을 추가했다.
- 적용한 변경 사항: PWA manifest, `/sw.js`, `/api/push/public-key`, `/api/push/subscriptions`, push subscription 저장소, web-push automation provider, 관리자 알림 설정의 `이 기기에서 알림 받기` UI 추가.
- 검증 결과: `npm run build` 통과, `npx tsc --noEmit` 통과, 로컬 프로덕션 서버에서 manifest/public-key/service-worker 200 응답 확인.
- 남은 위험 요소: Vercel CLI 로그인이 현재 환경에서 확인되지 않아 배포 환경변수 등록은 아직 직접 완료하지 못했다.
- 다음 단계: Vercel 프로젝트 환경변수에 `NEXT_PUBLIC_VAPID_PUBLIC_KEY`, `VAPID_PRIVATE_KEY`, `VAPID_SUBJECT`를 등록하고 재배포한 뒤 휴대폰에서 알림 권한을 테스트한다.
